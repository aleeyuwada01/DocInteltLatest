# DocIntel — System Architecture & Tech Stack

## Tech Stack Overview
- **Frontend Framework:** React 19 + Vite + TypeScript
- **Styling:** Tailwind CSS 4.0
- **Icons:** Lucide React
- **Notifications:** Sonner
- **Backend / BaaS:** Supabase (PostgreSQL 17, Auth, Storage)
- **File Uploads:** `tus-js-client` (Resumable chunked uploads)
- **AI Processing:** 
  - LlamaParse (Document extraction)
  - Google Gemini 2.5 Flash (OCR, Chat, Semantic Search, Content Summarization, Document Comparison)
  - text-embedding-004 (Google) for high-dimensional semantic mapping.

## Core Systems

### 1. Database Schema (Supabase)
The database uses strict Row Level Security (RLS) to ensure multi-tenant isolation.
- `profiles`: Links to Supabase Auth (`auth.users`). Defines `role`, `storage_limit`, and `parent_id` (for Department sub-accounts).
- `folders`: Hierarchical folder structure (`parent_id`).
- `files`: The core asset table. Tracks `storage_path`, raw `markdown`, `parsed_markdown`, `ai_description`, and `parsing_status`. 
- `embeddings`: Links to `files`. Stores `1536-dim` vectors generated by Gemini for semantic search.
- `chat_sessions` & `chat_messages`: Persists AI chat conversations.
- `search_history`: Persists global semantic search queries.

### 2. Authentication & Departments
- **Standard Sign-in:** Handled directly by `supabase.auth.signInWithPassword`.
- **Department Sub-accounts:** Managed by Admins via `/api/departments`. This endpoint uses the `SUPABASE_SERVICE_ROLE_KEY` to bypass restrictions, creating an auto-confirmed user in Auth, and inserting a `profiles` row with `parent_id` pointing to the Admin. 

### 3. Upload & Parsing Pipeline
1. **Frontend (TUS):** User selects file(s)/folder. The `UploadProgress` component manages the queue. `tus-js-client` uploads chunks to Supabase Storage (`/uploads` bucket).
2. **Database Trigger / Webhook:** Once a file hits Supabase Storage, a Supabase Database Webhook fires, sending a payload to the `/api/webhook` Vercel function.
3. **Backend Parsing (Vercel API):**
   - The webhook creates a `files` row in the DB with `parsing_status = 'parsing'`.
   - Based on MIME type, the file is either sent to LlamaParse (for PDFs/Word docs) or Gemini Vision (for Images).
   - Raw markdown is generated.
4. **Backend Intelligence (Vercel API):** 
   - Gemini processes the markdown to create an `ai_description`.
   - Gemini generates `1536-dim` embeddings.
   - DB is updated with `parsing_status = 'completed'` or `'error'`.
5. **Frontend Polling:** `MainContent.tsx` polls the `files` table for status changes and clears the loading spinners.

### 4. Vercel Serverless Functions
Local development uses a custom `server.ts` proxy to mirror Vercel's edge architecture. All APIs live in `/api/*.ts` and handle Auth headers explicitly using the Supabase Admin SDK or standard Client.

- `/api/chat.ts` - Streaming AI chat with continuous memory and citations.
- `/api/search.ts` - Consolidated endpoint for Vector semantic search, Related Files, and **Public Share Token Validation**.
- `/api/compare.ts` - Multi-document evaluation engine.
- `/api/webhook.ts` - LlamaParse/Gemini ingestion pipeline (structural extraction + AI enrichment).
- `/api/departments.ts` - Administrative sub-account management.
